Q: Rust?
A: I think Rust is an interesting language. The concept of it is not new, it has been explored in Haskell and other functional languages. Differences are, that Haskell is lazy, has a different and unusual syntax and uses other terminology than other procedural/imperative languages. People started to hype around Rust and from the past we know: the bigger the hype around a language, the faster it changes and the faster it dies. All in all: A big no, if I want memory-safety, I can use Haskell, it is a mature functional language. And if I wanna do low-level or imperative stuff, I can use C.

Q: Text editor/IDE?
A: I don't use IDE's. For me, a plain editor is enough.
In day-to-day life I use GNU/Emacs as my main editor and for quickly changing config files I usually use vi, joe or vim. Sometimes, if I need to paste some things from the clipboard into a file, I also use ed.
Q: Why not an IDE?
A: IDE's are usually increadibly huge bundles of software. I neither like to build these from sources nor download Gigabytes worth of precompiled binaries. Another point is, that when I write code, I start to have the design and naming convention of variables and function names in my head. By forcing yourself to not use an IDE which has got autocompletion and "IntelliSense", as M$ calls it, you enforce a consistent naming convention and coding style because you need a way to remember all these things. Furthermore, I like to think about how I want to design my software, which the current software development market discourages with it's rapid application development cycles. Using a plain text editor makes you think more carefully about what you do.
